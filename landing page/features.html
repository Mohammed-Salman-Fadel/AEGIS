Features

A local AI stack with orchestration built in.

AEGIS combines a Rust engine, CLI, web UI, local inference providers, Python RAG, and user-approved tools into one desktop workflow.

Core capabilities

Designed around what AEGIS actually does.

The system is not just a chat box. It is a local orchestration layer that keeps session state, model control, retrieval, documents, projects, and tools moving through the same backend contract.

01

Local inference providers

Route generation through Ollama or LM Studio while keeping provider selection, model listing, warmup, and switching behind the engine.

Ollama / LM Studio
02

Document-grounded RAG

Imported documents are indexed by the Python RAG service and scoped to the active conversation so context does not leak across sessions.

PDF / TXT / Chroma
03

Coding project context

Attach a local project folder, summarize supported files, and prepare patch-style code changes only after the user grants edit approval.

Project mode / patches
04

CLI that feels alive

Use sessions, slash-style tools, model commands, status checks, doctor, import flows, and live Markdown token streaming from the terminal.

Rust CLI / SSE
05

Web UI for sessions

Manage conversations, tools, project cards, model settings, metrics, document context, exports, light/dark mode, and Markdown rendering.

React / localhost
06

Local personalization

Store profile notes and response style preferences locally so the model can adapt without requiring an AEGIS account or remote profile.

User profile / styles

Architecture

A localhost stack with one routing brain.

The Rust engine exposes the shared HTTP boundary used by the CLI and web UI. It builds prompts, manages sessions, routes model calls, and delegates retrieval to the Python RAG service when document context is active.

01

User sends a prompt

The CLI or web app forwards the request to the Rust engine.

02

Engine assembles context

Session history, profile notes, selected mode, documents, and project context are prepared.

03

Provider generates locally

Ollama or LM Studio streams tokens back through the same engine endpoint.

04

Interface renders live

Markdown, code blocks, metrics, and tool feedback appear incrementally.

Privacy model

Secure by locality, transparent by design.

AEGIS is designed around localhost services and explicit user choices. Some setup steps may download models or dependencies, but normal inference, retrieval, sessions, and tools are intended to run locally.
